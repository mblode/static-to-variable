# glyph-forge (app) — agent instructions

Internal audit UI. Most routes consume static SVGs + manifest JSON generated by `@static-to-variable/glyph-forge-engine`; those routes should not hit `packages/variable-gen/` at request time. The `/generate` route is the exception: it is an operator workflow that writes ignored job state under `.pipeline-jobs/` and launches the CLI pipeline in an isolated temp checkout.

## Run

```bash
npm run forge:build                             # from repo root — regen cache
turbo dev --filter=@static-to-variable/studio           # https://static-to-variable.localhost
npm --workspace @static-to-variable/studio run generation:smoke
```

Dev server is fronted by **portless** (global install) — `portless static-to-variable next dev` proxies `https://static-to-variable.localhost` onto a random port in 4000-4999. No port collisions.

`predev` and `prebuild` run `scripts/sync-cache.ts`, which mirrors `packages/glyph-forge-engine/public-cache/svg/` and `packages/glyph-forge-engine/manifests/broken-glyphs.json` into `public/`. Both are `.gitignore`d.

## Routes

- `/` — glyph grid (filter bar: family, feature, verdict, free-text)
- `/g/[family]/[name]` — loupe view: 2×8 weight strip, overlay toggle, sidebar
- `/interventions` — operator handoff workspace for failed pipeline stages
- `/triage` — staged edit review queue
- `/generate` — file-dump generator for Circular sources → variable TTF job artifacts

## Conventions

- **No shadcn / no base-ui / no blode-icons** here. This is an internal tool — keep the dep graph small. Use plain Tailwind + the `cn()` utility in `lib/utils.ts`.
- **Server components by default**. The manifest JSON is loaded server-side via `lib/data.ts`; client components are opt-in (`"use client"`) for interactivity only (filters, overlay toggle).
- **SVG cells are plain `<img>` tags**. They're small, cacheable, and let the browser do lazy loading for free.
- **Route param encoding**: glyph names containing `.` and `_` are safe in URLs; `/` in glyph names (there are none now, but defensive) should be encoded.
- **Generator jobs are local and single-user**. Keep uploads, logs, and outputs inside `.pipeline-jobs/`; serve downloads only through registered artifact IDs.
- **Generator source modes**: default mode overlays uploaded files onto workspace templates/triage in an isolated temp repo. Strict mode must include all 16 Circular OTFs, both Glide `.glyphs` sources, and `circular-triage.json`.

## Gotchas

- **Don't commit `public/svg/` or `public/broken-glyphs.json`** — generated, gitignored.
- **Don't commit `.pipeline-jobs/`** — generated upload/job/output state, gitignored.
- **Port 3333** not 3000 — leaves the default Next.js port free for anything else running locally.
- **Missing SVGs render as broken images on purpose**: if a glyph doesn't exist in one of the 16 cells (e.g., the donor lacks `.ss02` variants), the cell shows a dim placeholder. This is a real audit signal, not a bug.
- **Portless cert lives in the system trust store**: if `https://static-to-variable.localhost` throws a cert error, run `portless trust` once. Safari needs `portless hosts sync` for `/etc/hosts` resolution.

## Do-not

- Do not add analytics, telemetry, or any third-party script. Internal tool.
- Do not read from `packages/variable-gen/` at runtime outside `/generate`; all audit-viewer data flows through the `glyph-forge-engine` manifest.
- Do not import directly from `packages/glyph-forge-engine/` via relative paths. Use the `@static-to-variable/glyph-forge-engine` workspace package for typed imports.
